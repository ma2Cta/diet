# ダイエット運用リポジトリ

## 目標
- 体重: 120kg → まず110kg → 最終100kg未満
- 運動（エアロバイク）は「今は続いていない前提」でOK。まず食事で落とす。
- 失敗しても戻れる設計（完璧主義を捨てて70点を継続）

## 前提条件（会話で確定）
- 1日3食
- 朝は時間がない（調理なし）
- 1回あたり調理＋片付け20分以内
- ホットクック（1人用）あり → 週2回のまとめ調理
- ご飯200gは「ちょうどいい」
- プロテインは美味しくなくて続かない → 食事からタンパク質確保
- 甘いもの欲求が強い（ストレス起因・仕事中に来る）
- 暴発トリガーは「コンビニ」「Uber（ついで注文）」
- 辛い系はそこまで / にんにくOK / 洋食寄りが好み
- Uber Eatsは「どうしても我慢できない時のみ」許容

## 使い方
1. `plans/base/strategy.md` を読んで運用ルールを固定する
2. `plans/base/weekly_plan.md` の通りに回す（朝固定・昼固定・夜ローテ）
3. `shopping/templates/shopping_list_week1.md` でネットスーパー発注
4. `logs/daily/` に `YYYY-MM-DD.md` を1日1ファイルで記録（詳細）
5. `logs/daily_log.csv` を1行更新（一覧管理）
6. 週末に `logs/weekly_summary.md` を更新（振り返り）

## 献立と買い物リストの運用（CSV）
1. `data/recipes/` のマスターを更新（レシピID・材料）
2. `plans/weekly/<YYYY-MM-DD>/meal_plan.csv` に週の献立を記入（週開始日）
3. 献立と `data/pantry/` の在庫をもとに `shopping/generated/` を作成
4. 発注内容を `shopping/orders/<YYYY-MM-DD>_order_final.csv` に確定

## 最重要（この設計の核）
- 糖質カット・甘味完全禁止はしない（反動で崩れる）
- 「脂質×砂糖」(チョコ/菓子パン/クリームアイス) を避け、甘味は低脂質で管理する
- コンビニ/Uberは“行動を禁止”せず、買うもの/頼むものを固定する
