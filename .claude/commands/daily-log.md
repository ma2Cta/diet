# 日次ログ記録

引数: 日付（例: `2026-03-03`）

対象日: $ARGUMENTS

## 実行手順

1. `logs/daily_log.csv` を読み、既存エントリを確認する。
2. `plans/weekly/` から対象日を含む週の `meal_plan.csv` を読み、予定献立を確認する。
3. ユーザーから提供された実食情報（体重・実食内容・備考）を元に `logs/daily/$ARGUMENTS.md` を作成する。
   - 情報が未提供の場合は、記録できる項目のみ記載してユーザーに確認を促す。
4. `logs/daily_log.csv` に1行追記する（日付が重複する場合は更新）。
5. 変更ファイルとコミットメッセージをユーザーに提示し、承認を得てからコミットする。
   - コミットメッセージ例: `log: add daily entry $ARGUMENTS`

## 完了後の出力

- 記録した内容のサマリー
- 予定との乖離があれば指摘
- 次アクション提案（週次振り返りのタイミングなど）
